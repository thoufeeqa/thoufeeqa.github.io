R0ver R0ver

A memory game

Overview

This was an early project, built in Unity

I took part in a game jam with a time limit of eight hours and a theme centered around memory games.

The aim is to manoeuvre a rover across a terrain with obstacles placed randomly. The catch however, is that you need to pre-program a set of instructions to the rover before it starts moving; a nod to how people at mission control have to pre-program instructions to execute, due to the amount of time it takes for signals to travel from the Earth to the Moon...or Mars.

The player is given 3 lives per level and the obstacles on each level are procedurally generated. The player explores the terrain and has to memorise the correct sequence of instructions to give (Right, Left or Forward) as they progress and get to the end of the stage.

I don't know how to animate that well, so each of the axles in the rover are rotated using code instead and the rover body is parented to these axles. So when it's turning, there's basically one section of the code manipulating the rover's overall movement, and another section of code that's rotating the axles at the same time.
